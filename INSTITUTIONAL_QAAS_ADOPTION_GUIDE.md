## 📋 The 5 Closure Gates

Every gap must pass through **5 validation gates**:

### **1. Artifact Gate**
```
Question: Does the artifact exist?
Example: "foundational_paper_package" exists with:
  ├─ Manuscript draft
  ├─ Reproducibility bundle
  ├─ Claim-boundary appendix
  └─ Submission tracker
```

### **2. Owner Gate**
```
Question: Is there a named, accountable owner?
Example: Scientific lead owns peer_review gap
  ├─ Name: [Person]
  ├─ Responsibility: Manuscript through publication
  ├─ Escalation: [Governance]
  └─ Timeline: [Target date]
```

### **3. Acceptance Criteria Gate**
```
Question: Are the criteria objectively measurable?
Example: Benchmark standardization requires:
  ✅ External-style dimensions documented
  ✅ Local benchmark commands mapped
  ✅ Translation layer implemented
  ✅ Validation passed
```

### **4. Claim-Boundary Gate**
```
Question: Does the claim boundary protect against overstatement?
Example: "Data provenance" claim boundary:
  ✅ What we claim: FAIR manifest exists
  ✅ What we don't claim: External system access validated
  ✅ What we defer: Long-term archival by external FAIR provider
```

### **5. Validation Hook Gate**
```
Question: Is there an automated validation?
Example: "fair_manifest_check" validates:
  ✅ Findable IDs present
  ✅ Access rules defined
  ✅ Interop schema valid
  ✅ Reuse license specified
  ✅ Checksum metadata correct
```
