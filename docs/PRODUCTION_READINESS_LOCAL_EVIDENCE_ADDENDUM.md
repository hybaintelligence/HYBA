# HYBA_FULLSTACK Local Evidence Readiness Addendum

**Status:** active addendum to `docs/PRODUCTION_READINESS.md`  
**Purpose:** define the production-readiness path when hosted CI cannot be used  
**Scope:** HYBA_FULLSTACK as HYBA's self-financing operating substrate

## 1. Decision

Hosted CI is preferred, but it is not the only valid source of release evidence.

When hosted CI cannot be completed, HYBA_FULLSTACK may use a local command-room evidence gate, provided the gate is run from a clean checkout by an accountable operator and the generated evidence packet is preserved with the release record.

This does not lower the standard. It moves the evidence source from hosted CI to controlled local execution.

## 2. New local gate scripts

The repo now exposes:

```bash
npm run prod:local:gate
npm run prod:live:gate
npm run prod:command-room:gate
```

These commands use `scripts/local_production_gate.py`.

## 3. RC local gate

Run:

```bash
git pull origin main
npm ci
npm run prod:local:gate
```

The RC gate records:

- live-deployment forensic audit;
- runtime mock/static telemetry guard;
- TypeScript lint;
- production build;
- backend unit tests;
- backend E2E tests;
- deployment E2E tests;
- deployment property tests.

It writes a JSON evidence packet under:

```text
artifacts/production_readiness/
```

## 4. Live environment gate

After production environment variables and pool credentials are loaded from a controlled local environment or secret store, run:

```bash
npm run prod:live:gate
```

The live gate records:

- production environment validation;
- operator credential shape;
- pool-profile completeness;
- required production flags;
- PULVINI live-cut state invariants when a state file is available.

## 5. Evidence packet requirements

A local evidence packet must be retained with:

- timestamped JSON output from the local gate;
- the git commit SHA tested;
- redacted environment summary;
- operator name or role;
- approval ticket or decision note;
- bridge health output after start;
- backend readiness output after start;
- authenticated mining status before any operator action;
- pool-side evidence if a live-pool window is initiated.

## 6. Launch boundary

Local evidence may authorize a controlled launch path, but it does not authorize unsupported claims.

HYBA may state that HYBA_FULLSTACK passed a local command-room readiness gate if the evidence packet passes.

HYBA must not claim revenue, solvency, payroll coverage, office-cost coverage, accepted shares, or stable hashrate until live pool evidence confirms those facts.

## 7. Operational sequence

1. Run `npm run prod:local:gate`.
2. Load production environment from a controlled source.
3. Run `npm run prod:live:gate`.
4. Start production with automatic mining disabled.
5. Check bridge health and backend readiness.
6. Log in as an authorized operator.
7. Check mining status before any action.
8. Start mining only by explicit operator action.
9. Preserve local and pool-side evidence.
10. Escalate treasury reliance only after repeated measured evidence.

## 8. Bottom line

HYBA_FULLSTACK can proceed without hosted CI only if command-room local evidence replaces it. The replacement is not verbal assurance; it is a timestamped, reproducible JSON evidence packet generated by the repo itself.
