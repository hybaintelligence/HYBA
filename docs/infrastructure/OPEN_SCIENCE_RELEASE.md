## Release Tiers

### Tier 1: Core Algorithms (Public Immediately)
```
What:       Quantum mathematics proofs + code
When:       Immediately upon completion
License:    MIT (derivative work permitted)
Rationale:  Science advances through transparency
Examples:   PULVINI proof, φ-resonance analysis

Policy:
  ✅ Publish to GitHub (public repo)
  ✅ Register DOI on Zenodo
  ✅ Announce publicly
```

### Tier 2: Research Evidence (Public + Peer Review)
```
What:       Reproducibility bundles, benchmarks, test data
When:       After peer review acceptance
License:    CC-BY 4.0 (attribution required)
Rationale:  Validated before wide dissemination
Examples:   Fresh-clone test results, benchmark comparisons

Policy:
  ✅ Peer review first
  ✅ Then publish to repository
  ✅ Link to peer review publication
```

### Tier 3: Customer Data (Confidential)
```
What:       Customer-specific results, proprietary runs
When:       Never (except customer permission)
License:    Private
Rationale:  Customer trust + competitive advantage
Examples:   Bank portfolio results, pharma screening outcomes

Policy:
  🔒 Encrypted storage (separate from public repo)
  🔒 No publication without consent
  🔒 Audit trail of access
```

### Tier 4: Experimental (Internal Only)
```
What:       Work-in-progress, failed experiments
When:       May publish later (after completion)
License:    TBD
Rationale:  Avoid false negatives in literature
Examples:   Failed proof attempts, optimization experiments

Policy:
  🔒 Private GitHub branches
  ⏳ Archive if no longer pursuing
  ✅ Can publish if completed later
```

## License Choices

### MIT License (Code)
```
Permits:     Commercial use, modification, distribution
Requires:    License and copyright notice
Best for:    Algorithms, libraries
Example:     PULVINI compression code
```

### CC-BY 4.0 (Data + Documentation)
```
Permits:     Commercial use, modification, distribution
Requires:    Attribution to source
Best for:    Research data, benchmarks, papers
Example:     φ-resonance evidence datasets
```

### CC0 1.0 (Public Domain)
```
Permits:     All uses without restriction
Requires:    Nothing
Best for:    Foundational datasets
Example:     Benchmark baseline results
```

## Redaction Rules

### What Gets Redacted?
```
❌ Customer names (unless they permit)
❌ Proprietary hardware details (if partner)
❌ Unreleased patents (pre-filing)
❌ Third-party confidential data
✅ Scientific results (with redactions above)
```

### Redaction Process
```
1. Research team proposes release
2. Legal review (60 minutes typical)
3. Patent attorney sign-off (if new IP)
4. Board approval (if material)
5. Publish with redactions documented
```

## Governance Approvals

### For Tier 1 (Public Immediately)
```
Approval: Author self-signs (researcher responsible)
Review: Scientific advisory board optional feedback
Timeline: Same day
```

### For Tier 2 (After Peer Review)
```
Approval: Peer reviewers approve publication
Review: Board confirmation of release
Timeline: After peer review cycle (2-3 months)
```

### For Tier 3-4 (Confidential/Internal)
```
Approval: CEO/Founder oversight
Review: Legal review if publishing later
Timeline: On-demand (no automatic release)
```

## Publication Examples

### Example 1: Publish Algorithm
```
GitHub:    PULVINI compression code pushed (MIT license)
Zenodo:    Full package + DOI registered
Twitter:   "We published PULVINI algorithm. Cite: DOI:10.5281/..."
Academic: Submitted to IEEE Transactions + Zenodo DOI link
```

### Example 2: Publish Research Data
```
GitHub:    Benchmark results dataset (CC-BY license)
Zenodo:    Metadata + data with DOI
Paper:     Cite dataset in publication
Replicators: Download data, reproduce our results
```

### Example 3: Keep Customer Data Private
```
GitHub:    Benchmark results published (anonymized)
Customer:  "Your results stay private"
Academic:  "Can we publish with your permission?" (request)
Result:    Customer chooses (most say yes with attribution)
```
