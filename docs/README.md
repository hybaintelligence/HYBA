# HYBA Fullstack Documentation

## Document Structure

This documentation follows McKinsey project organization standards with clear hierarchical categorization for efficient navigation and auditability.

### Executive Summary
High-level overviews and key findings for stakeholders and decision-makers.
- Location: `executive-summary/`

### Strategic Analysis
Long-term planning, roadmaps, and strategic direction documents.
- Location: `strategic-analysis/`

### Governance
Policy, authorization, and structural governance documentation.
- Location: `governance/`

### Implementation
Deployment plans, integration verification, and implementation roadmaps.
- Location: `implementation/`

### Evidence & Reviews
Audit trails, production readiness evidence, and review documentation.
- Location: `evidence/`

### Scientific Research
Mathematical foundations, quantum mining research, and theoretical frameworks.
- Location: `scientific/`

### Technical Reports
Capability registries, comparative analysis, and technical assessments.
- Location: `technical-reports/`

### Deployment Guides
Operational runbooks, deployment procedures, and cloud deployment documentation.
- Location: `deployment/` and `runbooks/`

### Compliance
Regulatory compliance and legal documentation.
- Location: `compliance/`

### Meeting Notes
Progress tracking and meeting documentation.
- Location: `meeting-notes/`

### Appendices
Supporting artifacts, implementation patches, and reference materials.
- Location: `appendices/`

### Special Projects
- `al_haras/` - Specialized project documentation


### Canonical Documentation Map

Start with [`CANONICAL_DOCUMENTS.md`](CANONICAL_DOCUMENTS.md) before adding or restoring documentation. It identifies the maintained source of truth for production readiness, deployment, governance, scientific, testing, and operations material, and it records legacy root-level documents that were removed during documentation cleanup.

## Navigation

For production readiness gates and deployment procedures, refer to the `deployment/` and `evidence/` directories.
For scientific foundations and mathematical certificates, refer to the `scientific/` directory.
For governance and authorization documentation, refer to the `governance/` directory.
