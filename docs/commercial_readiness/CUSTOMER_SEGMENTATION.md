## Ideal Customer Profiles (ICP)

### ICP 1: Research Institution
```
Company type: University/lab (Caltech, MIT, Oxbridge, etc.)
Budget: $1-2K/mo (research discretionary budget)
Pain: Quantum simulator queue overbooked
Buying trigger: Need to validate algorithm before hardware access
Qualification: "Do you have unpublished quantum algorithms?"
ROI: Time-to-publication reduced 3-6 months
```

### ICP 2: Financial Services
```
Company type: Bank/hedge fund (JP Morgan, Goldman, Citadel)
Budget: $50K+/mo (enterprise budget)
Pain: Quantum advantage ROI uncertain, NISQ unreliable
Buying trigger: CRO approves quantum research program
Qualification: "Do you have quantum ML research pipeline?"
ROI: Risk mitigation + deterministic portfolio optimization
```

### ICP 3: Pharma/Biotech
```
Company type: Drug discovery shop (Merck, Novartis, Genentech)
Budget: $20K-100K/mo (R&D budget)
Pain: Drug discovery cycles long (10+ years)
Buying trigger: Quantum computing could accelerate molecular sim
Qualification: "Do you simulate molecular dynamics?"
ROI: 10-30% acceleration on pharma screening
```

### ICP 4: Quantum Startups
```
Company type: Seed/Series A quantum startup
Budget: $5-10K/mo (stretch)
Pain: Can't build without quantum hardware
Buying trigger: Building quantum-native product
Qualification: "Are you building quantum algorithms?"
ROI: Proof of concept before hardware access
```

## Qualification Gates

### Gate 1: Problem Fit
```
Question: "Are you struggling with current quantum simulation?"
YES → Proceed
NO → Not qualified, suggest general marketing
```

### Gate 2: Budget
```
Question: "Can you commit $100+/mo for 3 months?"
YES (Starter/Professional) → Proceed
$50K+ (Enterprise) → Enterprise sales process
NO → Defer 6 months
```

### Gate 3: Use Case Validation
```
Question: "Is your use case research OR production critical?"
RESEARCH → Starter tier (fast onboarding)
PRODUCTION → Enterprise tier (governance required)
NEITHER → Not qualified
```

## Pilot Program Protocol

### Week 1: Onboarding
```
1. Account creation + API key generation
2. Welcome packet (docs, tutorials, examples)
3. Kickoff call (30 min technical overview)
4. Initial workload: "Hello HYBA" verification
```

### Week 2-4: Pilot Execution
```
1. Customer runs own algorithm on HYBA
2. Generates reproducibility evidence
3. Weekly check-in calls (progress review)
4. Optimization coaching (if needed)
```

### Week 5: Pilot Completion
```
1. Final results documented
2. ROI worksheet completed (time saved, cost avoided)
3. Decision point: Continue or not?
4. If continue: Upgrade tier or negotiate Enterprise contract
```
