# HENDRIX-Φ Structured Solver Naming Doctrine

## Decision

The public algorithm name is:

```text
HENDRIX-Φ Structured Solver
```

Expanded form:

```text
Harmonic Entropic Nonce Discovery via Resonant Icosahedral eXploration
```

Short operational form:

```text
HENDRIX-Φ
```

## Why rename away from Phi-Grover

`Phi-Grover` is useful as lineage language, but it creates a reviewer trap.

Grover is associated with unstructured needle-in-a-haystack search. HYBA's solver claim is different: the nonce space is treated as structured, geometrically covered, phi-resonant, memory-compressed, and guided by the M_32 dodecahedron/icosahedron manifold.

The algorithm should therefore not be named as if it were merely Grover with a parameter change. The correct public framing is a structured resonance solver with Grover ancestry, not a generic Grover implementation.

## Backward compatibility

The following names remain valid as aliases for migration and historical lineage:

```text
phi_grover
Phi-Grover
phi-Grover
structured Grover
```

They should not be the primary public or reviewer-facing name.

Code may keep internal compatibility aliases where changing them would break scripts, tests, manifests, or existing evidence packets. New documentation, commands, packets, and public claims should prefer `HENDRIX-Φ`.

## Personal dedication

The name honours Hendrix/Hendricks, the operator's dog, in the same cultural tradition that made Schrodinger's cat a permanent scientific image.

The code spelling should be `HENDRIX`, not `Hendrick's`, to avoid brand confusion while preserving the personal dedication.

```text
Schrodinger had the cat.
HYBA has HENDRIX.
```

## Reviewer-safe language

Use:

```text
HENDRIX-Φ is HYBA's structured phi-resonant nonce solver.
It has Grover ancestry but is not presented as unstructured Grover search.
It uses M_32 geometric coverage, phi-resonant scoring, PULVINI memory compression, field-coherence guidance, and measured proof-of-work verification.
```

Avoid:

```text

Quantum entanglement is present in the classical implementation.
Phi is magic because it is beautiful.
The solver has found a share before accepted-share evidence exists.
```

## Evidence-first rule

HENDRIX-Φ may be ambitious in architecture, but every production claim remains evidence-gated:

```text
local measured runtime
structured-vs-control benchmarks
phi ablation
memory-compression ablation
accepted-share evidence
pool-side confirmation
forensic packet hash
```

If evidence is absent, the system reports absence. It must never simulate success.
