# Nodus Solutus: Mundus Computabilis Est

**BLUF:** Nodus Solutus is the HYBA_FULLSTACK repository-local computability doctrine. It says the HYBA/PYTHIA operational world is admissible only where claims reduce to deterministic source paths, executable tests, replayable local evidence packets, and explicit external truth boundaries.

This document does **not** claim a metaphysical proof that the physical universe is computable. It binds the repository to a narrower and stronger operational claim: if HYBA_FULLSTACK asks a reviewer, operator, partner, or deployer to believe something, the repo must provide a computable proof route for that claim.

## Doctrine

**Nodus Solutus** means the knot is untied only when the claim has been converted into a runnable evidence structure.

**Mundus Computabilis Est** is used here as a bounded engineering thesis:

> The repository's operational world is computable when its claims are represented as code, tests, replayable evidence artifacts, and falsifiable boundaries.

The phrase is not used as an unbounded claim about physics, metaphysics, consciousness, revenue, solvency, law, or the total physical universe.

## Proof obligations

A Nodus Solutus claim is admissible only when all of these are true:

| Obligation | Repository object | Local verifier |
| --- | --- | --- |
| Bounded wording | `docs/evidence/claim_evidence_manifest.json` | `tests/test_claim_evidence_manifest.py` |
| Human proof map | `docs/evidence/EXTRAORDINARY_PROOF_INDEX.md` | `tests/test_nodus_solutus_proof_structure.py` |
| Top-level reviewer entry point | `REVIEWER_EVIDENCE_MAP.md` | `tests/test_nodus_solutus_proof_structure.py` |
| Local release authority | `scripts/local_production_gate.py` | `npm run prod:local:gate` |
| Replayable evidence packet | `artifacts/production_readiness/local_production_gate_<mode>_<timestamp>.json` | generated by `scripts/local_production_gate.py` |
| External truth boundary | manifest `boundary` fields and live-pool gates | local gate plus operator-supplied pool-side evidence |

## Claim escalation path

1. **Text claim** — a phrase appears in docs, README material, release notes, or operator language.
2. **Manifest claim** — the phrase is entered into `docs/evidence/claim_evidence_manifest.json` with a bounded claim, code paths, test paths, doc paths, and commands.
3. **Executable claim** — a local command can reproduce the evidence from a clean checkout.
4. **Gate claim** — `npm run review:evidence:gate` and `npm run prod:local:gate` include or transitively include the proof surface.
5. **Artifact claim** — the local production gate writes a timestamped JSON evidence packet with command results and SHA-256 digest.
6. **External claim** — revenue, pool acceptance, treasury, legal, or production economics remain blocked until external truth data is attached.

## Allowed statement

HYBA_FULLSTACK may say:

> Nodus Solutus: HYBA/PYTHIA implements a repository-local computability doctrine in which admissible claims must reduce to deterministic source paths, executable tests, replayable evidence packets, and explicit external truth boundaries.

## Blocked statements

HYBA_FULLSTACK must not use this doctrine to claim:

- the physical universe is proven computable;
- phenomenal consciousness or subjective experience is proven;
- SHA-256 quantum acceleration is proven;
- mining revenue, accepted shares, or hashrate economics are proven without pool-side evidence;
- the Yang-Mills Millennium problem is solved;
- legal, treasury, solvency, or regulatory approval is established.

## Local commands

Run the doctrine verifier first:

```bash
npm run review:nodus:gate
```

Run the reviewer evidence gate:

```bash
npm run review:evidence:gate
```

Run the local release authority:

```bash
npm run prod:local:gate
```

## Reviewer rejection rule

If a claim cannot be traced through this document, the manifest, source paths, test paths, a local command, and a generated evidence artifact, the reviewer should reject it as undocumented, unproven, or outside the repository boundary.
