# HYBA Sovereign Procurement Positioning

## Positioning statement

HYBA is a sovereign-deployable intelligence and analytics platform for organisations that require AI, data ingestion, optimisation, decision intelligence, and governance controls to operate inside customer-controlled infrastructure.

Unlike cloud-first platforms that depend on managed telemetry, remote policy engines, cloud billing, or third-party storage, HYBA can run under a local sovereign control plane that enforces deployment mode, data residency, source restrictions, usage governance, administrative privileges, and audit evidence inside the customer boundary.

## One-line version

```text
HYBA brings AI-grade intelligence into sovereign, regulated, and air-gapped environments without requiring customer data to leave the room.
```

## Buyer-safe claim ladder

### Conservative claim

HYBA supports configurable cloud, private-cloud, on-premise, sovereign-site, and air-gapped deployment profiles with local policy and audit controls.

### Stronger claim

HYBA can enforce local data-residency, ingestion, usage, admin, and audit policies without requiring cloud billing, cloud storage, external ingestion, or remote telemetry in sovereign modes.

### Highest safe claim

HYBA provides the technical control plane required for sovereign-grade deployment assessment: local policy enforcement, deployment attestation, privileged-admin dual control, local usage governance, and append-only audit evidence.

## Objection handling

### Is this accredited for classified deployment?

No automatic accreditation is claimed. HYBA provides the technical enforcement surfaces required to support the customer's accreditation process: local execution, local policy, local audit, data-residency enforcement, privileged-admin controls, and deployment attestation.

### Does HYBA require cloud?

No. HYBA can be configured for on-premise, sovereign-site, and air-gapped operation where cloud storage and external network ingestion are disabled by default.

### Can administrators bypass restrictions?

Privileged administration can require authorised roles, reason codes, second approval, or change-ticket binding, with every decision written to local audit evidence.

### How do we verify the system's posture?

HYBA exposes deployment attestation, policy decisions, evidence seals, and audit hash-chain events for customer inspection.
