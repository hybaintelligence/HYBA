## Product surfaces

### 1. Portfolio Evidence Compressor

**Purpose:** Turn massive portfolio, transaction, collateral, Sukuk lifecycle, or model-output evidence into compact replayable packets.

**HYBA substrate:** PULVINI memory compression with retained reconstruction kernels.

**Finance value:** Model validation teams, auditors, and regulators do not need opaque model dumps. They need a compact evidence packet with lineage, reconstruction proof, hash stability, exception list, and human review trail.

**Output:** compressed evidence manifest, kernel inventory, replay hash, reconstruction telemetry.

### 2. Adversarial Scenario Search Engine

**Purpose:** Search for hidden fragility in bounded financial state spaces.

**HYBA substrate:** Quantum-style deterministic search, dodecahedral basis, density-matrix discipline, numeric guards.

**Finance value:** Useful for liquidity stress, collateral waterfall breaks, Sukuk drift, Basel model-challenge, operational outage paths, and concentration-risk scenarios.

**Output:** candidate stress scenario, path hash, instability warning, human review queue entry.

### 3. Phi-Scaled Governance Scheduler

**Purpose:** Deterministically rank which evidence packets or anomalies require immediate human attention.

**HYBA substrate:** Golden-ratio scaling for deterministic weighting, folding, and resonance measurement.

**Finance value:** Compliance and model-risk teams drown in alerts. HYBA can prioritise without pretending phi alone proves truth.

**Output:** review priority score, normalised weight vector, ranking rationale, drift threshold.

### 4. Model-Risk Memory Layer

**Purpose:** Preserve why the system raised a concern, what evidence it used, what assumptions were active, and whether later outcomes confirmed or weakened the concern.

**HYBA substrate:** Persistent AI memory tables, memory snapshots, reasoning traces.

**Finance value:** Gives SR 11-7-style model governance, DORA-style ICT evidence, and audit teams something inspectable across time.

**Output:** reasoning trace, assumption lineage, memory snapshot, evidence retrieval log.

### 5. Intelligence Control Room

**Purpose:** Monitor internal finance-governance system health.

**HYBA substrate:** Metacognitive telemetry: pressure, exhaustion, confidence, prediction error, recovery mode, self-awareness, active events.

**Finance value:** Converts model governance from static review into live operational resilience. When confidence degrades or pressure spikes, the system escalates to human review.

**Output:** escalation signal, control-room telemetry, review-mode switch recommendation.

### 6. Proof-Obligation Ledger

**Purpose:** Make finance claims prove themselves before they are used commercially.

**HYBA substrate:** Millennium-problem operationalisation: proof discipline, invariant definition, counterexample search, falsification attempts.

**Finance value:** Every high-impact claim becomes a sealed obligation: assumptions, invariants, counterexamples, evidence sufficiency, independent reviewer field.

**Output:** claim register, proof obligation, counterexample challenge, falsification status, evidence sufficiency rating.
