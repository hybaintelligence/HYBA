# Cloudflare Pages Hardened Configuration Reference

This document provides the complete hardened configuration for HYBA_FULLSTACK on Cloudflare Pages.

## Dashboard Settings

Apply these settings in the Cloudflare Pages dashboard for the `hyba-fullstack` project:

```yaml
project: hyba-fullstack
production_branch: main

build:
  command: npm run build
  output_directory: dist
  root_directory: /
  build_system_version: 3

environment:
  variables: {}
  # Note: Set HYBA_BACKEND_URL in dashboard secrets if needed
  # HYBA_BACKEND_URL: https://<your-fastapi-origin>
```

## Configuration Validation

### Critical: Build Command

**DO USE:**
```bash
npm run build
```

**DO NOT USE:**
```bash
npm build
```

The command `npm build` is not a valid npm CLI command and will fail before Vite starts. The repository build script is defined in `package.json` as `scripts.build`, and deployment aliases (`cloudflare:build`, `pages:build`, `build:ci`) all delegate to `npm run build`.

### Validation Script

Run locally to verify build configuration integrity:

```bash
node scripts/assert_cloudflare_pages_build_config.mjs
```

## Build Output

- **Output Directory**: `dist` (generated by Vite)
- **Files Published**: All contents of `dist/` plus `public/_headers` and `public/_redirects`

## Security Headers and Routing

The deployment automatically includes:

### Security Headers (`public/_headers`)
- HSTS (Strict-Transport-Security)
- X-Frame-Options (frame denial)
- X-Content-Type-Options (MIME sniffing protection)
- Referrer-Policy (strict)
- Permissions-Policy (locked-down)

### Routing Rules (`public/_redirects`)
- SPA fallback routing for `index.html`
- `/api/*` routes preserved for Cloudflare Pages Functions
- `/health/*` routes preserved for health checks
- `/bridge/health` for edge bridge status

## Pages Functions

Cloudflare Pages Functions (in `functions/` directory) provide:

- API proxy to `HYBA_BACKEND_URL` for `/api/*` routes
- Health check proxying for `/health/*` routes
- Edge-side filtering and authentication

## Deployment Checklist

Before connecting a branch to Cloudflare Pages:

- [ ] Local build succeeds: `npm run build`
- [ ] Build validation passes: `node scripts/assert_cloudflare_pages_build_config.mjs`
- [ ] Linting passes: `npm run lint`
- [ ] Tests pass: `npm run test:backend && npm run test:e2e:backend`
- [ ] Dashboard build command is exactly: `npm run build`
- [ ] Dashboard output directory is exactly: `dist`
- [ ] Dashboard root directory is: `/`
- [ ] Build system version is: 3

Run all checks with:
```bash
npm run prod:check
```

## Post-Deployment Validation

After a successful deployment to Cloudflare Pages, verify:

1. **Edge Bridge Status**
   ```
   GET https://<app-domain>/bridge/health
   ```
   Should return Cloudflare edge bridge status and backend reachability.

2. **Backend Readiness**
   ```
   GET https://<app-domain>/api/health/readiness
   ```
   Should proxy to FastAPI backend readiness endpoint.

3. **UI Health Indicator**
   - Header latency pill should report "connected" state
   - Verify after first successful telemetry poll

## Troubleshooting

### Build Fails Immediately
- Check that build command is exactly `npm run build`, not `npm build`
- Run local build: `npm run build`
- Check for missing Node modules: `npm install`

### 404 on API Routes
- Verify Pages Functions are deployed: check `functions/` directory
- Confirm environment variable `HYBA_BACKEND_URL` is set
- Test backend connectivity locally before deployment

### No Immutable Asset Caching
- Verify `public/_headers` is present and deployed
- Check that Vite hash fingerprinting is working: look for `.hash` in asset filenames
- Rebuild and redeploy if headers are not present

### SPA Routing Not Working
- Verify `public/_redirects` is deployed
- Confirm `/api/*` and `/health/*` routes are excluded from fallback
- Test with a non-existent route to trigger fallback behavior
