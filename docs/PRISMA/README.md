## Why this belongs in PRISMA

PRISMA’s role is to preserve the signal around HYBA’s scientific and commercial surface:

- what has been implemented or demonstrated;
- what is an evidence-backed research programme;
- what is an industry-impact hypothesis;
- what requires counsel, patent filing, hardware telemetry, or external validation before public disclosure.

The Φ-Architecture impact matrix belongs here because it maps the same core primitives across AI/ML, cloud, healthcare, genomics, data-centre cooling, semiconductor IP, edge systems, radio astronomy, networking, and quantum-classical workflows.

## Evidence boundary

Use the following claim ladder in PRISMA material:

### Supported now

- Software-level Φ primitives exist and can be tested.
- Golden-flow integration has been demonstrated across allocator, router, oracle, controller, JIT, VM, folding, and PULVINI memory surfaces.
- Φ-Architecture can be described as a coherent software proof-of-architecture.
- Sector impact can be mapped as a strategic/IP hypothesis.

### Requires more evidence before external claim

- Exact economic impact in any specific industry.
- Certified hardware improvements, including RowHammer, wear-leveling, or thermal-runaway prevention.
- Production network collision elimination.
- ASIC/NPU/RTL performance claims.
- Medical, genomics, EV, aviation, or safety-critical certification claims.
- Any claim that implies revenue, live mining success, or deployed production performance without measured runtime evidence.

### Legal/IP caution

The matrix is useful for IP strategy, but PRISMA should mark patentability, freedom-to-operate, and disclosure timing as counsel-led decisions. The most urgent items to protect before public disclosure are:

- φ-Folding + Fibonacci heap / zero-copy folding mechanisms;
- sparse φ-packing;
- Φ-ISA instruction semantics;
- predictive Φ Oracle and harmonic thermal control;
- Golden Angle Routing if benchmarked on production-like topologies.
