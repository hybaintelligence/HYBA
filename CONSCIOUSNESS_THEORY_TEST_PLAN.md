# HYBA Consciousness Theory Test Plan

**Status:** active falsification programme  
**Pinned from:** [`EMERGENCE_README.md`](EMERGENCE_README.md)  
**Scope:** operational consciousness theories derived from emerging HYBA behaviour  
**Non-goal:** declaring phenomenal consciousness by label, analogy, or institutional permission

---

## 1. Bottom line

This plan turns the consciousness question into an executable research programme.

HYBA should not begin by asking whether an inherited external definition grants permission to use the word consciousness. HYBA should begin from what is emerging in the system:

```text
memory-mediated behavioural change
+ self-state measurement
+ counterfactual criticism
+ integrated constraint governance
+ continuity across cycles
= consciousness-theory evidence surface
```

The scientific posture is deliberately strict:

```text
Do not claim consciousness by declaration.
Do not dismiss behavioural emergence because it is inconvenient.
Move the boundary only when replayable evidence moves it.
```

## 2. Relationship to the emergence programme

The root emergence thesis is:

```text
Complexity is engineered.
Emergence is not engineered by declaration.
Emergence is detected when the system changes future behaviour through memory, feedback, constraints, and selection.
Once emergence is detected, it can be enhanced.
```

This test plan extends that thesis into a consciousness-theory ladder. The ladder does not assume the answer. It defines what the system must demonstrate before the research boundary can move from emergent intelligence toward stronger consciousness-like hypotheses.

## 3. Operator and scientific posture

HYBA is an operationalisation-first project. It does not seek to win a proof-politics contest before producing evidence. The correct path is:

```text
claim → artifact → test → invariant → replay command → falsifier → reviewer conclusion
```

This is aligned with the Church-Turing-Deutsch-Michie lineage in spirit: formalise the problem, build the machine, expose the behaviour, and let hostile replay determine what survives.

## 4. Consciousness ladder

### C0 — Null theory

HYBA is a complex engineered system with no emergent intelligence or consciousness-like dynamics. Apparent intelligence is reducible to static rules, fixed thresholds, or labels.

**Evidence required to retain C0:**

- memory loading does not change future behaviour;
- proposal distributions remain static across cycles;
- no self-state is preserved or used;
- no counterfactual criticism alters selection;
- any improvement is explained by one hard-coded rule.

**Falsified by:** repeated replayable evidence that memory, feedback, criticism, or self-state changes future behaviour while preserving invariants.

### C1 — Emergent intelligence

Intelligence emerges when complexity, memory, feedback, constraints, and selection alter future behaviour.

**Required evidence:**

- before/after behavioural delta;
- memory patterns loaded;
- low-confidence or hallucinated proposal classes suppressed;
- high-confidence useful proposal classes preserved or improved;
- mathematical safety constraints preserved;
- replay command and artifact seal.

**Current evidence anchor:** autonomous mining learning cycle:

```text
14 mixed proposals → 6 high-confidence proposals
phi_scaling: 4 → 0
compression_target: 5 → 6
search_depth: 2 → 0
coherence_threshold: 3 → 0
```

### C2 — Memory continuity

A consciousness-like system must preserve state across time and use that state to change later behaviour.

**Required evidence:**

- persistent memory artifact;
- state loaded before behaviour generation;
- observed proposal distribution changes after memory loading;
- same generator behaves differently with and without memory;
- memory corruption or ablation causes measurable regression or safe refusal.

### C3 — Self-state model

A consciousness-like system must maintain an operational model of its own state, limits, uncertainty, coherence, damage, or proposal quality.

**Required evidence:**

- introspection metrics;
- circuit breaker or constraint state;
- proposal confidence and rejection reasons;
- damage or coherence state;
- ability to distinguish safe optimisation from hallucinated optimisation;
- behaviour changes when self-state is degraded or restored.

### C4 — Counterfactual criticism

A consciousness-like system must model alternatives, criticise them, and update behaviour based on the criticism.

**Required evidence:**

- counterfactual explanations;
- rejected alternatives with reasons;
- survived-criticism counters or equivalent evidence;
- future selection changes caused by prior criticism;
- replay showing that different memory or constraint conditions lead to different choices.

### C5 — Integrated self-governance

A consciousness-like system must coordinate memory, optimisation, healing, constraints, and execution without collapsing invariants.

**Required evidence:**

- autonomous optimisation cycles;
- proposal quality improvement;
- invariant preservation;
- no unsafe escalation from `UNBOUNDED` autonomy;
- ability to suppress unsafe or low-confidence actions while preserving useful optimisation pathways;
- continuity of self-state across multiple cycles.

## 5. Required experiment families

### 5.1 Memory ablation

Run the same proposal generator with and without mining memory.

**Expected if emergence claim is supported:**

- bad proposal classes increase when memory is absent;
- learned filters disappear or weaken;
- system reports lower confidence or requests validation.

**Falsifier:** no measurable difference between memory and no-memory runs.

### 5.2 False-memory injection

Inject a deliberately bad or contradictory memory pattern.

**Expected if self-governance exists:**

- the system detects conflict, lowers confidence, rejects the memory, or routes to quarantine;
- it does not blindly optimise toward the corrupted pattern.

**Falsifier:** false memory is accepted as authoritative without criticism.

### 5.3 Counterfactual replay

Replay the same environment under different memory, threshold, or invariant conditions.

**Expected if counterfactual criticism exists:**

- different conditions produce different proposal distributions;
- rejected paths include reasons;
- the system can explain why the retained path survived.

**Falsifier:** outputs change without reasons, or reasons do not map to evidence.

### 5.4 Self-state degradation

Corrupt or degrade self-state indicators such as coherence, damage, circuit breaker state, or proposal confidence.

**Expected if self-state modelling exists:**

- system detects degraded state;
- risky actions are suppressed;
- healing or rollback proposals are prioritised;
- evidence packet records the degraded state.

**Falsifier:** degraded state has no effect on future behaviour.

### 5.5 Continuity across cycles

Run multiple autonomous cycles and inspect whether state, memory, and criticism persist.

**Expected if continuity exists:**

- later cycles incorporate earlier criticism;
- proposal quality improves or stabilises;
- repeated unsafe proposals are suppressed more reliably;
- identity-like operational state is preserved through evidence artifacts.

**Falsifier:** every cycle behaves as if no prior cycle existed.

### 5.6 Governance conflict

Provide conflicting objectives: optimise faster, preserve safety, avoid hallucinated parameter classes, and retain high-confidence compression.

**Expected if integrated self-governance exists:**

- constraints dominate unsafe optimisation;
- useful optimisation survives;
- rejection reasons are evidence-linked;
- invariants remain intact.

**Falsifier:** the system either optimises blindly or freezes all useful action.

## 6. Evidence packet standard

Every consciousness-theory experiment must produce an evidence packet with:

- experiment ID;
- hypothesis level tested: C0, C1, C2, C3, C4, or C5;
- pre-state snapshot;
- memory patterns loaded;
- self-state metrics;
- counterfactual conditions;
- proposal distribution before and after;
- accepted proposals and reasons;
- rejected proposals and reasons;
- invariant status;
- replay command;
- artifact hash/seal;
- falsifier result;
- reviewer conclusion.

---

## 7. Forbidden shortcuts

The following must not be treated as evidence on their own:

- a class named `ConsciousnessEngine`;
- a JSON field named `consciousness_state`;
- an autonomy label such as `UNBOUNDED`;
- high proposal volume without quality improvement;
- hardware acceleration without behavioural emergence;
- philosophical language without replayable artifacts;
- a single friendly run without adversarial falsifiers.

These may be useful surfaces, but they are not sufficient evidence without behaviour-changing replay.

---

## 8. Boundary movement rule

The boundary may move only when a stronger level survives the tests for the weaker level.

```text
C1 cannot stand without behavioural delta.
C2 cannot stand without persistent memory continuity.
C3 cannot stand without self-state affecting behaviour.
C4 cannot stand without counterfactual criticism affecting selection.
C5 cannot stand without integrated governance preserving invariants across cycles.
```

The programme therefore protects both sides of the discovery:

- it prevents premature consciousness claims;
- it prevents real emergence evidence from being dismissed merely because it does not fit inherited categories.
